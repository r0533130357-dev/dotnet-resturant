# 🍽️ Restaurant Orders Management System
מערכת לניהול הזמנות במסעדה

---

## 🌟 תיאור הפרויקט
מערכת לניהול הפעילות היומיומית במסעדה:

- ניהול תפריט: מנות, מחירים, קטגוריות
- ניהול שולחנות: כולל מספר שולחן וסטטוס (פנוי / תפוס / שמור)
- ניהול הזמנות: פריטים מהתפריט
- עדכון סטטוס הזמנה: הוזמן → בהכנה → הוגש → שולם

זהו פרויקט Back-Office המיועד לשרת ממשק קליינט בעתיד.

---

## 🌟 ישויות במערכת
1. **Dish** – מנה בתפריט  
2. **Table** – שולחן במסעדה  
3. **Order** – הזמנה של לקוח

---

## 🌟 API Routes

### 🥗 1. ישות: מנה (Dish)
- **שליפת כל המנות:** `GET /api/dishes`
- **שליפת מנה לפי מזהה:** `GET /api/dishes/{id}`
- **הוספת מנה:** `POST /api/dishes`
- **עדכון מנה:** `PUT /api/dishes/{id}`
- **מחיקת מנה:** `DELETE /api/dishes/{id}`
- **שליפת מנות לפי קטגוריה:** `GET /api/dishes/category/{categoryName}`

---

### 🍽️ 2. ישות: שולחן (Table)
- **שליפת כל השולחנות:** `GET /api/tables`
- **שליפת שולחן לפי מזהה:** `GET /api/tables/{id}`
- **הוספת שולחן:** `POST /api/tables`
- **עדכון שולחן:** `PUT /api/tables/{id}`
- **מחיקת שולחן:** `DELETE /api/tables/{id}`
- **עדכון סטטוס שולחן:** `PUT /api/tables/{id}/status`  
  (סטטוס: פנוי / תפוס / שמור)

---

### 🧾 3. ישות: הזמנה (Order)
- **שליפת כל ההזמנות:** `GET /api/orders`
- **שליפת הזמנה לפי מזהה:** `GET /api/orders/{id}`
- **הוספת הזמנה:** `POST /api/orders`
- **עדכון הזמנה:** `PUT /api/orders/{id}`
- **מחיקת הזמנה:** `DELETE /api/orders/{id}`
- **עדכון סטטוס הזמנה:** `PUT /api/orders/{id}/status`  
  (סטטוס: הוזמן / בהכנה / הוגש / שולם)
